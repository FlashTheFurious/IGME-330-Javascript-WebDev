Proposal

  1. Link to the JikanJS Wrapper by Mateo Aranda Documentation

Major Features -

  1. Anime Search - Allows users to search for animes by filtering the results on the basis of number of results, name and genre.
  2. Add to List - Allows the user to build a temporary list of anime names by adding searched animes to this list.
  3. Save Favorites - Allows the user to save their favorite list of animes and share it with the Community. Anime Likes can be found in the Community page

Noteworthy

  1. Fetch was used to get Json Data
  2. Bulma is used to make the site mobile friendly
  3. 4 Web Components are used to follow DRY code principles
  4. Uses local storage to save UI state
  5. Stores favorites in local storage
  6. Uses Firebase to store and retrieve data on how many likes an anime has recieved, i.e, how many times it was favorited and shared

Grading

  1. Ajax via fetch() is on the App page
  2. App page has required number of controls
  3. UI state is saved and restored on the App page, except for the genre field
  4. Cards have a favorite button and a remove from favorites button
  5. The Favorite system works and has been tested on banjo and a local server
  6. Firebase functionality is used to store likes data of animes
  7. There is an embedded font

  11. All images were properly optimized
